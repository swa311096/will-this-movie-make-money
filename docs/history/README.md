# History of LLMs

This section exists for one reason: modern AI terms make more sense when you know what problem each phase was trying to solve.

The goal is not to memorize a timeline of papers or company launches.

The goal is to understand:

- what limitation existed
- what changed
- why that change mattered
- which modern concepts came out of it

## Teaching Standard

This section should be written as if someone is learning the topic for the first time.

That means each chapter should explain:

- the problem in plain English
- the intuition behind the approach
- a simple example
- what improved
- what still did not work
- why that led to the next phase

If a chapter reads like a list of terms, it is not finished yet.

## How To Read This Section

Read these parts in order:

1. `01-before-transformers/README.md`
2. `02-transformers-and-scaling.md`
3. `03-from-chat-models-to-ai-systems.md`
4. `04-how-to-read-ai-claims.md`

The first part is now broken into smaller chapters because it contains many foundational concepts that are easy to gloss over too quickly.

## What You Should Get Out Of This

By the end of this section, you should be able to read phrases like:

- "1M-token context window"
- "Mixture-of-Experts"
- "instruction-tuned model"
- "RAG pipeline"
- "agent memory"

and understand what those claims are actually pointing at.
