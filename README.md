# AI Learning Lab

This repository is a learning-first AI project. The goal is not to collect random AI demos. The goal is to build a single example project that forces the right concepts to become concrete:

- how LLM apps work
- when to use retrieval
- how tools and agents differ
- how MCP fits in
- how skills and agent instructions work
- how to write evals
- how to talk about these topics clearly

## What This Repo Contains

- `docs/00-overview.md`: mental model for the whole system
- `docs/01-learning-modules.md`: exact learning curriculum
- `docs/02-project-blueprint.md`: the example project we will build
- `docs/03-roadmap.md`: phase-by-phase implementation plan

## How We Will Work

We will use one repository for both theory and implementation.

Why:

- concepts stick better when linked to working code
- evals, prompts, skills, and architecture stay versioned together
- showcase value is stronger when explanation and implementation live side by side

## Project Direction

We will build an AI research and execution copilot. The system will start as a simple LLM app and gradually grow into:

- a structured-output assistant
- a retrieval-backed system
- a tool-using workflow engine
- an agent with controlled loops
- an MCP-integrated app
- a skills-aware system
- an evaluated and demoable project

## Success Criteria

By the end of this project, you should be able to:

- explain the difference between prompting, RAG, tools, agents, MCP, skills, and evals
- implement each of those concepts in code
- justify when to use each pattern and when not to
- demo the project as a coherent showcase instead of disconnected experiments
