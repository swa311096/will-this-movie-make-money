# Posts

This folder contains public post drafts derived from the learning and build process in this repository.

The goal is not to force content for the sake of content. The goal is to make public writing a natural output of real understanding.

## Post Workflow

For each major learning milestone:

1. understand the concept
2. build the concept into the project
3. verify it with an eval, test, or manual check
4. draft one short public post

## What A Good Post Should Do

A good post should:

- explain one clear idea
- connect the idea to something built in this repo
- avoid hype and vague claims
- be short enough to post publicly without losing the point

## Suggested Post Types

- concept explanation
- implementation update
- comparison post
- evals or reliability lesson

## Naming Convention

Use:

- `001-<topic>.md`
- `002-<topic>.md`

This keeps the learning history ordered.
